# Configuration Placeholders

Before using this project, replace all placeholders with your actual values.

## Required Placeholders

| Placeholder | Description | Example |
|---|---|---|
| `YOUR_AWS_ACCOUNT_ID` | Your 12-digit AWS account ID | `123456789012` |
| `YOUR_AWS_REGION` | AWS region to deploy in | `us-east-1`, `ap-south-1` |
| `YOUR_KMS_KEY_ID` | KMS key ID created by bootstrap | Auto-generated by bootstrap |
| `YOUR_TFSTATE_BUCKET` | S3 bucket for Terraform state | Auto-generated by bootstrap |
| `YOUR_BOOTSTRAP_BUCKET` | S3 seed bucket for bootstrap state | Create once: `aws s3 mb s3://YOUR_PROJECT_NAME-bootstrap-state` |
| `YOUR_GITHUB_ORG` | Your GitHub organization or username | `myorg` |
| `YOUR_PROJECT_NAME` | Your project name (used as prefix for resources) | `myproject` |
| `YOUR_SSO_ADMIN_ROLE` | AWS SSO Administrator role name | From AWS Console → IAM → Roles, filter by `AWSReservedSSO` |
| `YOUR_SSO_PLATFORM_ADMIN_ROLE` | EKS Platform Admin SSO role name | Create in AWS SSO or IAM Identity Center |
| `YOUR_SSO_DEVELOPER_ROLE` | EKS Developer SSO role name | Create in AWS SSO or IAM Identity Center |
| `YOUR_SSO_READONLY_ROLE` | EKS ReadOnly SSO role name | Create in AWS SSO or IAM Identity Center |

## Where Each Placeholder Appears

### AWS Account ID (`YOUR_AWS_ACCOUNT_ID`)
- `techitfactory-infra/environments/dev/main.tf` — KMS key ARN in backend
- `techitfactory-infra/bootstrap/github-oidc.tf` — ECR policy ARN
- `techitfactory-infra/environments/dev/eks-access.tf` — SSO role ARNs
- `techitfactory-app/.github/workflows/build-all.yml` — ECR registry URL
- `techitfactory-gitops/environments/*/*/kustomization.yaml` — All image references
- `techitfactory-gitops/apps/platform/cluster-autoscaler.yaml` — IRSA role ARN

### Terraform State Bucket (`YOUR_TFSTATE_BUCKET`)
- `techitfactory-infra/environments/dev/main.tf` — Backend config

> **Note:** This bucket is auto-created by the bootstrap. Run bootstrap once manually,
> then update this file with the output value `state_bucket_name`.

### Bootstrap Bucket (`YOUR_BOOTSTRAP_BUCKET`)
- `techitfactory-infra/bootstrap/backend.tf` — Backend for bootstrap state

> **Note:** Create this bucket manually ONCE before running bootstrap:
> ```bash
> aws s3 mb s3://YOUR_PROJECT_NAME-bootstrap-state --region YOUR_AWS_REGION
> ```

### GitHub Organization (`YOUR_GITHUB_ORG`)
- `techitfactory-infra/bootstrap/github-oidc.tf` — OIDC trust policy conditions
- `techitfactory-app/.github/workflows/ci.yml` — GitOps repo checkout
- `techitfactory-app/.github/workflows/build-all.yml` — GitOps repo checkout
- `techitfactory-app/.github/workflows/release.yml` — GitOps repo checkout
- `techitfactory-gitops/apps/root-app.yaml` — Source repo URL

### SSO Roles (`YOUR_SSO_*_ROLE`)
- `techitfactory-infra/environments/dev/eks-access.tf`

> **Note:** SSO roles are optional. If you do not use AWS IAM Identity Center,
> you can remove the `eks-access.tf` file entirely. EKS access entries are only
> needed if you want to restrict cluster access by SSO role.

## Quick Setup (Find & Replace)

Run this in your terminal to replace all placeholders at once:

```bash
# Set your values
ACCOUNT_ID="123456789012"
REGION="us-east-1"
GITHUB_ORG="myorg"
PROJECT_NAME="myproject"

# Replace in all files
find . -type f \( -name "*.tf" -o -name "*.yml" -o -name "*.yaml" -o -name "*.md" \) \
  ! -path "*/.git/*" ! -path "*/.terraform/*" | xargs sed -i \
  -e "s/YOUR_AWS_ACCOUNT_ID/${ACCOUNT_ID}/g" \
  -e "s/YOUR_AWS_REGION/${REGION}/g" \
  -e "s/YOUR_GITHUB_ORG/${GITHUB_ORG}/g"
```

> After running bootstrap, update `YOUR_TFSTATE_BUCKET` and `YOUR_KMS_KEY_ID` with the
> actual output values from `terraform output`.

## GitHub Secrets Required

Add these secrets to each GitHub repository:

### techitfactory-infra (infrastructure repo)
| Secret | Value |
|---|---|
| `AWS_ROLE_ARN` | `arn:aws:iam::YOUR_AWS_ACCOUNT_ID:role/YOUR_PROJECT_NAME-github-terraform` |

### techitfactory-app (application repo)
| Secret | Value |
|---|---|
| `AWS_ROLE_ARN` | `arn:aws:iam::YOUR_AWS_ACCOUNT_ID:role/YOUR_PROJECT_NAME-github-terraform` |
| `GITOPS_TOKEN` | GitHub PAT with `repo` scope on the gitops repo |
| `SONAR_TOKEN` | From sonarcloud.io (optional) |
| `SLACK_WEBHOOK_URL` | Slack incoming webhook URL (optional) |

## Bootstrap Order

1. Create bootstrap bucket manually: `aws s3 mb s3://YOUR_PROJECT_NAME-bootstrap-state`
2. Replace all `YOUR_BOOTSTRAP_BUCKET` with `YOUR_PROJECT_NAME-bootstrap-state`
3. Run bootstrap locally: `cd techitfactory-infra/bootstrap && terraform init && terraform apply`
4. Copy outputs → update `YOUR_TFSTATE_BUCKET` and `YOUR_KMS_KEY_ID` in `environments/dev/main.tf`
5. Push to GitHub with `[deploy]` → CI/CD takes over from here
