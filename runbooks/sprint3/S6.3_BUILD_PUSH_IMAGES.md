# Sprint 3, Story 6.3: Build and Push Docker Images to ECR

## Story Summary
Build all 6 service Docker images locally, push them to ECR, update the GitOps
image tags, and let ArgoCD deploy them to the cluster.

**Full flow:**
```
Local Code  →  docker build  →  docker push  →  ECR
                                                   ↓
GitOps repo (update newTag)  →  ArgoCD detects  →  EKS pulls image  →  Pods start
```

> **This is the manual bootstrap step.** Sprint 5 (CI/CD) automates the entire
> build → push → update tag → ArgoCD sync pipeline with GitHub Actions.

---

## Prerequisites
- S6.1 completed (ArgoCD running)
- S6.2 completed (root-app applied, namespaces created)
- Docker running locally (`docker info` works)
- AWS SSO active (`aws sts get-caller-identity --profile techitfactory` works)
- `techitfactory-app` and `techitfactory-gitops` repos cloned locally

## Part 1: Set Variables

Run this block once — all subsequent commands use these variables.

```bash
export AWS_ACCOUNT="535002890483"
export AWS_REGION="ap-south-1"
export ECR_BASE="${AWS_ACCOUNT}.dkr.ecr.${AWS_REGION}.amazonaws.com/techitfactory"
export APP_DIR=~/Desktop/Devops-Project/techitfactory-app
export GITOPS_DIR=~/Desktop/Devops-Project/techitfactory-gitops
export IMAGE_TAG="v0.1.0"
```

---

## Part 2: Authenticate Docker to ECR

```bash
aws ecr get-login-password \
  --region ap-south-1 \
  --profile techitfactory \
| docker login \
  --username AWS \
  --password-stdin \
  ${AWS_ACCOUNT}.dkr.ecr.${AWS_REGION}.amazonaws.com
```

Expected: `Login Succeeded`

> ECR tokens expire after 12 hours — re-run this if you get auth errors later.

---

## Part 3: Build All Images

Each service has its own `Dockerfile`. Build all 6:

```bash
# Frontend (nginx static)
docker build -t ${ECR_BASE}/frontend:${IMAGE_TAG} ${APP_DIR}/services/frontend

# API Gateway
docker build -t ${ECR_BASE}/api-gateway:${IMAGE_TAG} ${APP_DIR}/services/api-gateway

# User Service
docker build -t ${ECR_BASE}/user-service:${IMAGE_TAG} ${APP_DIR}/services/user-service

# Product Service  (directory name is 'product', ECR name is 'product-service')
docker build -t ${ECR_BASE}/product-service:${IMAGE_TAG} ${APP_DIR}/services/product

# Order Service    (directory name is 'order', ECR name is 'order-service')
docker build -t ${ECR_BASE}/order-service:${IMAGE_TAG} ${APP_DIR}/services/order

# Cart Service     (directory name is 'cart', ECR name is 'cart-service')
docker build -t ${ECR_BASE}/cart-service:${IMAGE_TAG} ${APP_DIR}/services/cart
```

Verify all images were built:
```bash
docker images | grep ${AWS_ACCOUNT}
```

---

## Part 4: Push All Images to ECR

```bash
docker push ${ECR_BASE}/frontend:${IMAGE_TAG}
docker push ${ECR_BASE}/api-gateway:${IMAGE_TAG}
docker push ${ECR_BASE}/user-service:${IMAGE_TAG}
docker push ${ECR_BASE}/product-service:${IMAGE_TAG}
docker push ${ECR_BASE}/order-service:${IMAGE_TAG}
docker push ${ECR_BASE}/cart-service:${IMAGE_TAG}
```

Verify images exist in ECR:
```bash
aws ecr describe-images \
  --region ap-south-1 \
  --profile techitfactory \
  --query 'imageDetails[*].[repositoryName,imageTags[0]]' \
  --output table \
  $(for repo in frontend api-gateway user-service product-service order-service cart-service; do
      echo "--repository-name techitfactory/${repo}"
    done) 2>/dev/null

# Quick check — list all tags for one repo
aws ecr list-images \
  --repository-name techitfactory/api-gateway \
  --region ap-south-1 \
  --profile techitfactory \
  --query 'imageIds[*].imageTag' \
  --output table
```

## Part 5: Update Image Tags in GitOps Repo

Now tell ArgoCD which image tag to deploy by updating the kustomization files:

```bash
cd ${GITOPS_DIR}

# Update newTag in all 6 dev kustomization files
for service in frontend api-gateway user-service product-service order-service cart-service; do
  FILE="environments/dev/${service}/kustomization.yaml"
  sed -i "s/newTag: .*/newTag: ${IMAGE_TAG}/" ${FILE}
  echo "Updated ${FILE}"
done
```

Verify the changes:
```bash
grep -r "newTag" environments/dev/ --include="*.yaml"
# All should show:  newTag: v0.1.0
```

Commit and push to trigger ArgoCD:
```bash
git add environments/dev/
git commit -m "chore: bootstrap images at ${IMAGE_TAG} for all services"
git push origin main
```

## Part 6: Create Kubernetes Secrets

The `user-service` and `api-gateway` require `JWT_SECRET` — without it
`user-service` will exit immediately on startup.

```bash
kubectl create secret generic app-secrets \
  --from-literal=jwt-secret=$(openssl rand -base64 32) \
  -n techitfactory

# Save this — it must stay the same across redeploys
kubectl get secret app-secrets -n techitfactory \
  -o jsonpath="{.data.jwt-secret}" | base64 -d && echo
```

## Part 7: ArgoCD Syncs Automatically

ArgoCD polls Git every 3 minutes. To sync immediately:

```bash
# Sync app-of-apps (it syncs child applications)
argocd app sync root-app

# Optional: verify child apps exist and are healthy
argocd app list
```

Watch pods start:
```bash
kubectl get pods -n techitfactory -w
```

Expected — all pods `Running`:
```
NAME                               READY   STATUS    RESTARTS
api-gateway-xxx                    1/1     Running   0
cart-service-xxx                   1/1     Running   0
frontend-xxx                       1/1     Running   0
order-service-xxx                  1/1     Running   0
product-service-xxx                1/1     Running   0
user-service-xxx                   1/1     Running   0
```
