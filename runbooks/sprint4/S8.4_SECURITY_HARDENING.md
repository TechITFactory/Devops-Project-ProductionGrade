## Story Summary
Verify and demonstrate the security controls already baked into the platform.
Real-world projects need to show they're hardened — this story documents and
proves each control is working.

## Part 1: Verify Non-Root Containers

Every container should run as a non-root user (UID 1001). This is set in
`values.yaml` via `podSecurityContext`:

```bash
# Check which user each pod is running as
kubectl get pods -n techitfactory -o name | while read pod; do
  echo -n "${pod}: "
  kubectl exec -n techitfactory ${pod} -- id 2>/dev/null || echo "exec failed"
done
```

Expected: `uid=1001 gid=1001`

Also verify no container has privilege escalation:
```bash
kubectl get pods -n techitfactory -o json \
  | jq '.items[].spec.containers[].securityContext.allowPrivilegeEscalation'
# All should be: false
```

## Part 2: Verify Network Policies

NetworkPolicies restrict which pods can talk to which. Our policy allows:
- Pods in the **same namespace** to communicate (service-to-service)
- Pods in the **monitoring namespace** to scrape `/metrics`
- **All other ingress is denied**

```bash
# List network policies
kubectl get networkpolicies -n techitfactory
# Should show one policy per service

# Describe one to see the rules
kubectl describe networkpolicy api-gateway -n techitfactory
```

### Test that policies work

```bash
# This SHOULD work — api-gateway to product-service (same namespace)
kubectl exec -it deploy/api-gateway -n techitfactory -- \
  wget -qO- http://product-service/health && echo "OK — same namespace allowed"

# This SHOULD FAIL — trying to reach techitfactory from kube-system
kubectl run test-pod --image=busybox --rm -it --restart=Never -n kube-system -- \
  wget -qO- --timeout=5 http://api-gateway.techitfactory.svc.cluster.local/health \
  && echo "UNEXPECTED: should be blocked" \
  || echo "CORRECT: cross-namespace blocked"
```

## Part 4: ECR Image Scanning Results

ECR scans images automatically on push (`scan_on_push = true` in Terraform).
Review the results:

```bash
# Check scan results for each service
for repo in frontend api-gateway user-service product-service order-service cart-service; do
  echo "=== techitfactory/${repo} ==="
  aws ecr describe-image-scan-findings \
    --repository-name techitfactory/${repo} \
    --image-id imageTag=v0.1.0 \
    --region ap-south-1 \
    --profile techitfactory \
    --query 'imageScanFindings.findingSeverityCounts' \
    --output table 2>/dev/null || echo "Scan not complete yet"
done
```

### Acceptable Thresholds

| Severity | Action |
|----------|--------|
| CRITICAL | Must fix before prod |
| HIGH | Fix within 1 sprint |
| MEDIUM | Track and fix in next sprint |
| LOW / INFO | Informational — review |

> In Sprint 5, the CI pipeline will block merges if Trivy finds unfixed
> CRITICAL/HIGH vulnerabilities (`exit-code: '1'`, `ignore-unfixed: true`).

## Part 5: PodDisruptionBudgets — Zero-Downtime Maintenance

PDBs ensure at least 1 pod stays running when nodes are drained
(e.g. during EKS version upgrades):

```bash
# List all PDBs
kubectl get pdb -n techitfactory

# Expected: one PDB per service, minAvailable: 1
```

### Simulate a Node Drain

```bash
# Get a node name
NODE=$(kubectl get nodes -o name | head -1 | cut -d/ -f2)

# Drain the node (simulates maintenance)
kubectl drain ${NODE} --ignore-daemonsets --delete-emptydir-data --dry-run=client
# The --dry-run flag means nothing actually happens — just shows what would happen

# You should see: "evicting pod" for pods, but PDB protects the last replica
```

## Part 7: Security Summary

Run this quick audit script to get a pass/fail summary:

```bash
echo "=== TechITFactory Security Audit ==="

echo -n "1. Non-root containers: "
NONROOT=$(kubectl get pods -n techitfactory -o json \
  | jq '[.items[].spec.securityContext.runAsNonRoot] | all' 2>/dev/null)
[ "$NONROOT" = "true" ] && echo "PASS" || echo "FAIL"

echo -n "2. NetworkPolicies exist: "
COUNT=$(kubectl get networkpolicies -n techitfactory --no-headers 2>/dev/null | wc -l)
[ "$COUNT" -gt 0 ] && echo "PASS (${COUNT} policies)" || echo "FAIL"

echo -n "3. PodDisruptionBudgets exist: "
COUNT=$(kubectl get pdb -n techitfactory --no-headers 2>/dev/null | wc -l)
[ "$COUNT" -gt 0 ] && echo "PASS (${COUNT} PDBs)" || echo "FAIL"

echo -n "4. app-secrets exists: "
kubectl get secret app-secrets -n techitfactory &>/dev/null && echo "PASS" || echo "FAIL"

echo -n "5. No hardcoded passwords in gitops: "
FOUND=$(grep -r "adminPassword:" \
  ~/Desktop/Devops-Project/techitfactory-gitops --include="*.yaml" 2>/dev/null | wc -l)
[ "$FOUND" -eq 0 ] && echo "PASS" || echo "FAIL (${FOUND} occurrences found)"
```
