# Sprint 4, Story 8.1: Walking Skeleton Verification

## Story Summary
Prove the entire platform works end-to-end. All 6 services should be running,
healthy, and reachable. This is the "does it actually work?" checkpoint before
adding automation in Sprint 5.

**What we're verifying:**
- All pods Running and passing health checks
- ArgoCD shows all apps Synced + Healthy
- Traffic flows correctly: Browser → Frontend → API Gateway → Services
- Each service responds on its own endpoint
- Secrets are wired correctly (JWT auth works)

---

## Prerequisites
- S6.1–S6.3 completed (ArgoCD running, images built and pushed to ECR)
- `app-secrets` Kubernetes Secret created in `techitfactory` namespace
- `kubectl` and `argocd` CLI configured

---

## Part 1: Verify ArgoCD — All Apps Synced

```bash
argocd app list
```

Expected output — every app shows `Synced` and `Healthy`:
```
NAME             CLUSTER    NAMESPACE       PROJECT  STATUS  HEALTH
api-gateway      in-cluster techitfactory   default  Synced  Healthy
cart-service     in-cluster techitfactory   default  Synced  Healthy
frontend         in-cluster techitfactory   default  Synced  Healthy
order-service    in-cluster techitfactory   default  Synced  Healthy
product-service  in-cluster techitfactory   default  Synced  Healthy
user-service     in-cluster techitfactory   default  Synced  Healthy
```

If any app shows `OutOfSync` — force a sync:
```bash
argocd app sync <app-name>
```

If any app shows `Degraded` — check pod events:
```bash
kubectl describe pod -l app=<app-name> -n techitfactory
```

## Part 2: Verify All Pods Running

```bash
kubectl get pods -n techitfactory -o wide
```

All pods should be `1/1 Running` with `0` restarts. If any pod is:

| Status | Cause | Fix |
|--------|-------|-----|
| `ImagePullBackOff` | Image not in ECR or wrong tag | Re-check S6.3 — verify tag in kustomization.yaml |
| `CrashLoopBackOff` | App crashing on start | Check logs: `kubectl logs <pod> -n techitfactory` |
| `Pending` | No node capacity | Check: `kubectl describe pod <pod> -n techitfactory` |
| `Init:Error` | Init container failed | Check: `kubectl logs <pod> -c <init-container> -n techitfactory` |

Check `user-service` specifically — it exits if `JWT_SECRET` is missing:
```bash
kubectl logs -l app=user-service -n techitfactory --tail=20
# Should NOT contain "FATAL: JWT_SECRET environment variable is required"
```

## Part 3: Service Communication Map

```
Internet
    │
    ▼
[Frontend :80]          ← nginx serving static UI
    │  (JS calls /api/*)
    ▼
[API Gateway :3000]     ← single entry point, JWT verification
    ├──→ [User Service :3000]     POST /api/users/login
    ├──→ [Product Service :3000]  GET  /api/products
    ├──→ [Order Service :8000]    GET  /api/orders
    └──→ [Cart Service :3000]     GET  /api/cart
```

### Verify Services Can Reach Each Other

```bash
# Exec into api-gateway pod and test internal DNS
kubectl exec -it deploy/api-gateway -n techitfactory -- \
  wget -qO- http://product-service/health && echo

kubectl exec -it deploy/api-gateway -n techitfactory -- \
  wget -qO- http://user-service/health && echo

kubectl exec -it deploy/api-gateway -n techitfactory -- \
  wget -qO- http://order-service/health && echo

kubectl exec -it deploy/api-gateway -n techitfactory -- \
  wget -qO- http://cart-service/health && echo
```

Each should return: `{"status":"ok"}`

## Part 4: End-to-End Smoke Test

### Step 4.1: Port-Forward API Gateway
```bash
# Run in a separate terminal — keep it open
kubectl port-forward svc/api-gateway -n techitfactory 8000:80
```

### Step 4.2: Health Check
```bash
curl -s http://localhost:8000/health | jq
# Expected: {"status":"ok"}
```

### Step 4.3: Products
```bash
curl -s http://localhost:8000/api/products | jq
# Expected: array of product objects
```

### Step 4.4: User Registration + Login
```bash
# Register a user
curl -s -X POST http://localhost:8000/api/users/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"test@example.com","password":"Test1234!"}' | jq

# Login
TOKEN=$(curl -s -X POST http://localhost:8000/api/users/login \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"Test1234!"}' | jq -r '.token')

echo "Token: ${TOKEN}"
```

### Step 4.5: Authenticated Endpoints
```bash
# Get current user (requires JWT)
curl -s http://localhost:8000/api/users/me \
  -H "Authorization: Bearer ${TOKEN}" | jq

# Add to cart
curl -s -X POST http://localhost:8000/api/cart \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"productId":"1","quantity":2}' | jq

# View cart
curl -s http://localhost:8000/api/cart \
  -H "Authorization: Bearer ${TOKEN}" | jq

# Place order
curl -s -X POST http://localhost:8000/api/orders \
  -H "Authorization: Bearer ${TOKEN}" \
  -H "Content-Type: application/json" \
  -d '{"items":[{"productId":"1","quantity":2}]}' | jq
```

### Step 4.6: Frontend
```bash
# Port-forward frontend in a separate terminal
kubectl port-forward svc/frontend -n techitfactory 8080:80

# Open http://localhost:8080 in browser
```
