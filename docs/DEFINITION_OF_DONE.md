# Definition of Done (DoD)

A story is **DONE** when ALL of the following are true:

## Code
- [ ] Code is committed to `main` branch
- [ ] Code passes linting
- [ ] Code passes unit tests
- [ ] No hardcoded secrets

## Review
- [ ] Pull Request approved by at least 1 reviewer
- [ ] CI pipeline is green

## Documentation
- [ ] README updated if behavior changed
- [ ] Runbook updated if operational procedure changed

## Demo
- [ ] Feature can be demonstrated in Sprint Review

---

## Definition of Ready (DoR)

A story is **READY** to be worked on when:
- [ ] Acceptance Criteria are clear
- [ ] Story is estimated (story points assigned)
- [ ] No external blockers
