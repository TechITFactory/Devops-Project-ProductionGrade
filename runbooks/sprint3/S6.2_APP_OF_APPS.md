# Sprint 3, Story 6.2: Configure App-of-Apps Pattern

## Story Summary
Set up the App-of-Apps pattern so ArgoCD manages all applications declaratively from Git.

**What We're Creating:**
- Root Application (manages all child apps)
- ArgoCD Application manifests for all 6 services + monitoring
- Kustomize base + overlay structure for dev and prod

---

## Prerequisites
- S6.1 completed (ArgoCD running, CLI logged in)
- `techitfactory-gitops` repository cloned locally
- kubectl configured for your EKS cluster

## Part 2: Create Required Secrets

**Do this BEFORE applying the root-app** — services will crash if secrets are missing.

```bash
# Create the app-secrets secret (JWT, optional admin seed)
kubectl create namespace techitfactory   # only needed if cluster is fresh

kubectl create secret generic app-secrets \
  --from-literal=jwt-secret=$(openssl rand -base64 32) \
  -n techitfactory

# Optional: seed an admin user on first deploy
kubectl patch secret app-secrets -n techitfactory \
  --type=json \
  -p='[
    {"op":"add","path":"/data/seed-admin-email","value":"'$(echo -n "admin@techitfactory.com" | base64)'"},
    {"op":"add","path":"/data/seed-admin-password","value":"'$(echo -n "ChangeMe123!" | base64)'"}
  ]'
```

> **Important:** The `jwt-secret` must be the same value across all deployments.
> The seed password is only used on first boot to create the admin user.

## Part 3: Bootstrap ArgoCD with Root App

### Step 3.1: Apply Root App

```bash
cd ~/Desktop/Devops-Project/techitfactory-gitops

# This is the ONE command that starts everything
kubectl apply -f apps/root-app.yaml
```

### Step 3.2: Verify Root App is Syncing

```bash
argocd app get root-app

# Wait for it to become Synced
argocd app wait root-app --sync
```

ArgoCD now watches `apps/` and syncs all child applications automatically.

## Part 5: How Kustomize Overlays Work

Each service has:
- **base/** — shared K8s manifests (Deployment + Service)
- **environments/dev/** — dev-specific overlay (image tag, replicas, env patches)

### Example: dev/api-gateway/kustomization.yaml

```yaml
apiVersion: kustomize.config.k8s.io/v1beta1
kind: Kustomization

namespace: techitfactory

resources:
  - ../../../base/api-gateway

images:
  - name: IMAGE_PLACEHOLDER              # must match image name in base/
    newName: <AWS_ACCOUNT>.dkr.ecr.ap-south-1.amazonaws.com/techitfactory/api-gateway
    newTag: latest

replicas:
  - name: api-gateway
    count: 2

patches:
  - patch: |-
      - op: add
        path: /spec/template/spec/containers/0/env/-   # <-- note the /- suffix to APPEND
        value:
          name: ENVIRONMENT
          value: dev
    target:
      kind: Deployment
      name: api-gateway
```

> **Key:** Use `env/-` (with the trailing `-`) to **append** an env var.
> Using `env` without `-` would **replace** the entire env array, breaking existing vars.

## Part 6: Updating Image Tags (CI/CD flow)

When CI builds a new image, it updates the `newTag` in the overlay:

```bash
# This is done automatically by ci.yml, but you can do it manually:
cd techitfactory-gitops

# Update dev overlay for api-gateway
yq -i '.images[0].newTag = "abc1234"' environments/dev/api-gateway/kustomization.yaml

git add . && git commit -m "ci: Update api-gateway to abc1234"
git push

# ArgoCD detects the change and syncs automatically
```

## GitOps Workflow (The Full Loop)

```
Developer → Push to techitfactory-app
                   ↓
         CI (GitHub Actions) builds image
                   ↓
         CI pushes image to ECR
                   ↓
         CI updates newTag in techitfactory-gitops
                   ↓
         ArgoCD detects Git change
                   ↓
         ArgoCD syncs to EKS cluster
```
